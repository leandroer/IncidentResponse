Selection principle

Choose tools for the evidence source and decision at hand. Preserve original telemetry, record collection time and operator actions, and validate automated findings before containment or eradication.

TELEMETRY

Network, cloud, SaaS, and containers

Collect control-plane, identity, workload, packet, and application evidence as a single incident timeline.

Network

  • Wireshark — packet-level inspection; captures may contain credentials and regulated data.
  • Zeek — structured network metadata and protocol analysis.
  • Suricata — IDS/IPS, protocol logging, and rule-based detection.

Cloud and SaaS

  • Microsoft Defender for Cloud — posture and workload protection across Azure, AWS, and Google Cloud.
  • AWS CloudTrail and GuardDuty — control-plane evidence and managed threat detection.
  • Google Security Command Center — cloud findings, posture, and threat detection.
  • Microsoft Purview Audit — Microsoft 365 user and administrative activity.

Containers and Kubernetes

  • Microsoft Defender for Containers — multicloud Kubernetes posture, image vulnerability assessment, and runtime threat protection.
  • Falco — runtime detection using kernel and Kubernetes signals.
  • Trivy — image, filesystem, configuration, and SBOM assessment.
  • Kubernetes audit logs — API activity and control-plane evidence.

Evidence discipline

Record the source, collector, UTC timestamps, acquisition method, tool version, cryptographic hash, storage location, and every transfer. Avoid changing system state unless the response decision justifies it and the action is recorded.

ANALYSIS

Malware analysis

Separate initial classification, dynamic behavior, and reverse engineering. Never treat a single scanner verdict as conclusive.

Public-service handling warning

Before submitting a file, URL, packet capture, document, or indicator to a public analysis service, confirm the organization’s data-handling policy. Public or community submissions may become accessible to vendors, researchers, or other service users. Use private analysis, contractual controls, or an isolated internal sandbox for sensitive material.

  • VirusTotal — multi-engine enrichment and relationship analysis; submission visibility depends on service and license.
  • ANY.RUN — interactive dynamic analysis; verify privacy mode before uploading.
  • capa — identifies executable capabilities from static or dynamic evidence.
  • Ghidra and IDA Pro — disassembly and reverse engineering.
  • CAPE Sandbox — self-hosted automated malware analysis and configuration extraction.
